### Boundary Conditions

- **RESEARCH.md exists but is empty or under 50 words**: gather-synthesize.md synthesizer has a "50-word threshold" quality filter mentioned in research-workflow.md key_constraints. `has_research = true` in init.cjs because file exists, but lens-aware reuse logic needs to check minimum content too. Trigger: feature dir contains an empty or stub `RESEARCH.md`. — `/Users/philliphall/get-shit-done-pe/get-shit-done/bin/lib/init.cjs:491` (`files.some(f => f === 'RESEARCH.md'...`) — no size check

- **Gatherer output path directory missing**: `mkdir -p "${OUTPUT_DIR}/research"` in research-workflow.md Step 1 creates the directory, but if OUTPUT_DIR itself doesn't exist (e.g., feature directory was never created), mkdir succeeds by creating the full path. - **User returns with /enhance for SAME feature that already has /enhance research**: Exact lens match, research should be reused. Trigger: `RESEARCH.md` date < `FEATURE.md` last modified date. — `/Users/philliphall/get-shit-done-pe/get-shit-done/bin/lib/init.cjs:504-506` — research_path set, no modification-time check

### Known Issues in Ecosystem

- **Claude Code Task() invocation ambiguity**: LLM-as-runtime systems treat pseudo-code as suggestions unless the instruction is unambiguously imperative. The model's own diagnosis in the prior session (referenced in BRIEF.md) confirms "Invoke @" is treated as "delegate to primary collaborator or spawn one agent" rather than "orchestrate 6 parallel Task calls." No external URL — this is the documented root cause in `/Users/philliphall/get-shit-done-pe/.planning/capabilities/pipeline-execution/BRIEF.md:36`

- **Parallel Task spawning in long context**: When an orchestrator has accumulated significant context (from prior pipeline stages), spawning 6 parallel Task calls simultaneously can trigger context-limit errors on the orchestrator side before all tasks are dispatched. The gather-synthesize pattern mitigates this by passing paths not content, but the orchestrator still assembles a context payload in Step 2 before spawning. — [First principles: 6 simultaneous Task dispatches each require the orchestrator to hold the dispatch state in working memory; the context payload assembled in research-workflow.md Step 2 includes PROJECT.md + STATE.md + ROADMAP.md + CAPABILITY.md + FEATURE.md + framing context, which can be substantial]

- **AI instruction drift under sequential execution**: When a workflow references another workflow via `@` and both contain behavioral instructions, the model may "blend" the two instruction sets rather than treating them as separate execution contexts. This is the exact failure mode described in BRIEF.md for the "Invoke @" pattern — the model partially executes research-workflow.md inline rather than spawning it as a Task. — `/Users/philliphall/get-shit-done-pe/.planning/capabilities/pipeline-execution/BRIEF.md:23-24`
