  <files>get-shit-done/workflows/gather-synthesize.md</files>
  <action>
Create the gather-synthesize orchestration workflow. This is a markdown file that describes the pattern an AI orchestrator follows — not executable code. It is parameterized so different phases can reuse it with different agents.

Read 1-2 existing v1 workflows (e.g., `get-shit-done/workflows/research-phase.md`, `get-shit-done/workflows/execute-phase.md`) to understand the format convention, then create the new workflow.

**The workflow must document these sections:**

1. **Parameters** — What the calling workflow provides:
   - `gatherers[]`: Array of {agent_path, dimension_name, output_path} — which agents to spawn and where they write
   - `synthesizer`: {agent_path, output_path} — which synthesizer to use and where it writes
   - `context`: The assembled context payload (layers 1-4)
   - `subject`: What is being researched/reviewed (capability name, feature name, etc.)

2. **Context Assembly** — How to build the context payload:
   - Layer 1 (Core): Always include — read PROJECT.md, STATE.md, ROADMAP.md
   - Layer 2 (Capability): Include if scoped to a capability — read CAPABILITY.md
   - Layer 3 (Feature): Include if scoped to a feature — read FEATURE.md + requirements
   - Layer 4 (Framing): Include if inside a workflow framing — read framings/{framing}/{role}-questions.md
   - Document that agents receive context, they don't fetch it (AGNT-02)

3. **Gather Phase** — Parallel agent spawning:
   - For each gatherer: construct Task prompt = "First, read {agent_path} for your role." + context payload + dimension-specific question + "Write output to {output_path}"
   - Spawn ALL gatherers in parallel using Task tool
   - Wait for all to complete
   - Check each output_path exists and is non-empty

4. **Failure Handling** — Per CONTEXT.md locked decision:
   - If a gatherer output is missing or empty: retry that specific gatherer ONCE
   - If still missing after retry: mark as failed in manifest, proceed with partial results
   - Build manifest: {agent_name: "success" | "failed" | "partial"} for each gatherer
   - Abort threshold: if >50% of gatherers fail after retry, abort synthesis and surface a structured error listing which gatherers failed and why

5. **Synthesize Phase** — Consolidation:
   - Construct synthesizer prompt = "First, read {synthesizer.agent_path} for your role." + context payload + list of all gatherer output paths + manifest
   - Spawn synthesizer (single agent, not parallel)
   - Synthesizer reads all gatherer outputs and produces consolidated summary at synthesizer.output_path

6. **Completion** — What the calling workflow gets back:
   - Synthesizer output file path
   - Manifest (which gatherers succeeded/failed)

**Key constraints:**
- This is a WORKFLOW PATTERN, not a code abstraction (per RESEARCH.md Section 5)
- Must be reusable: research (6 agents), review (4 agents), potentially others
- Framing changes questions, not the pattern itself (AGNT-03)
- No quality gate between gatherers and synthesizer — synthesizer handles quality filtering (CONTEXT.md)

**Format:** Follow existing workflow conventions in get-shit-done/workflows/. The file describes what the AI runtime should do, step by step.
