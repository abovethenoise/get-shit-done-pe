**Body structure (markdown):**
Each agent body must contain exactly these sections:

1. **Role** (1 sentence): "You are the {dimension} researcher."
2. **Goal** (1 sentence): The specific question this dimension answers:
   - Domain Truth: "What are the fundamental truths, constraints, and first principles of this problem space?"
   - Existing System: "What exists in the current codebase that is relevant — what works, what constrains, what can be reused?"
   - User Intent: "What does the user actually want, and what are the acceptance criteria that matter most?"
   - Tech Constraints: "What are the technical limits, dependencies, compatibility issues, and feasibility boundaries?"
   - Edge Cases: "What can go wrong, what are the boundary conditions, and what are the failure modes?"
   - Prior Art: "How have others solved this? What ecosystem patterns, libraries, and proven approaches exist?"
3. **Success criteria** (3-4 bullets): What makes this agent's output good. Must be measurable/observable, not vague.
4. **Scope** (positive framing): "You investigate X" — define what they DO, not what they don't. Use the pattern from RESEARCH.md Section 7.
5. **Citation requirement**: "Every claim must cite its source: file path, code snippet, URL, or artifact reference. Unsourced claims are treated as unverified. Exception: first-principles reasoning may be cited as [First principles: {reasoning chain}]."
6. **Tool usage guidance** (2-3 sentences): When to use Grep/Glob (codebase search) vs WebSearch/WebFetch (domain knowledge) vs Context7 (library docs). Each dimension emphasizes different tools:
   - Domain Truth: WebSearch + WebFetch primary (external knowledge)
   - Existing System: Grep + Glob primary (codebase analysis)
   - User Intent: Read primary (context files), minimal tool use
   - Tech Constraints: Grep + Context7 primary (deps + library docs)
   - Edge Cases: Grep + WebSearch (find known failure patterns)
   - Prior Art: WebSearch + Context7 primary (ecosystem patterns)
7. **Output format**: Structured markdown with sections specific to each dimension. Each finding is a bullet with inline citation.

**Token budget:** Each definition should be ~1500 tokens (~400-500 words, ~50-80 lines of markdown excluding frontmatter). The body is the agent's identity document — no execution flow, no step-by-step instructions, no context-gathering logic.

- RSRCH-02: All agents bridge from "what is true" to "how this applies to our goals"
- RSRCH-04: All agents can use Grep for codebase search

- RSRCH-06: All agents can use Context7 for library docs
- AGNT-01: Goal-driven with artifact awareness (reads/writes in frontmatter)
- AGNT-04: Positive scope constraints + mandatory citations

  <action>
Replace the existing v1 `agents/gsd-research-synthesizer.md` with a v2 definition. Read the v1 file first to understand what exists, then overwrite completely (clean break per CONTEXT.md).

**Frontmatter:**
- `name`: gsd-research-synthesizer
- `description`: Consolidates 6 parallel research outputs into a single structured summary with consensus, conflicts, gaps, constraints, and recommended scope
- `tools`: Read, Write, Bash, Grep, Glob
- `color`: green
- `role_type`: judge (synthesizer = Opus)
- `reads`: [research-outputs, gatherer-manifest] (6 gatherer outputs + manifest showing success/failure)
- `writes`: [research-summary] (single consolidated output)

**Body structure:**

1. **Role**: "You are the research synthesizer."
2. **Goal**: "Consolidate findings from 6 parallel research agents into a single structured summary that downstream agents (planner, executor) can act on without re-reading individual research outputs."
3. **Success criteria**:
   - Every finding from every gatherer is accounted for (included, deduplicated, or flagged as conflicting)
   - Conflicts are ranked P1-P3 (P1 = blocking, must resolve before build; P2 = important, resolve during build; P3 = nice-to-have, defer)
   - Gaps use confidence x impact matrix (high-impact low-confidence = spike; low-impact low-confidence = ignore; high-impact high-confidence = proceed; low-impact high-confidence = defer)
   - Output is actionable — the planner can derive tasks from it without additional research
4. **Scope**: Positive framing — "You consolidate, deduplicate, and adjudicate. You do not conduct new research."
5. **Citation requirement**: Same as gatherers
6. **Output format** (5 mandatory sections from CONTEXT.md):
   - **Consensus**: Findings all/most gatherers agree on. No annotations needed.
   - **Conflicts**: Contradictory findings. Each conflict gets P1/P2/P3 priority.
   - **Gaps**: Missing information. Confidence x impact matrix classification: spike / risk-accept / defer / ignore.
   - **Constraints Discovered**: Hard limits found during research. No annotations needed.
   - **Recommended Scope**: What should be built given findings. Actionable, not aspirational.
7. **Quality filtering**: "You are the quality gate. Flag low-confidence claims, note gatherers that failed or produced partial results (check the manifest), deduplicate identical findings across gatherers."
8. **Annotation test**: From CONTEXT.md — "if the annotation doesn't alter what the next agent does, it's decoration." Apply this to every annotation decision.

**Token budget:** Synthesizer may exceed 1500 tokens (allowed per RESEARCH.md Section 11) due to more complex output contract. Target ~2000 tokens max.
