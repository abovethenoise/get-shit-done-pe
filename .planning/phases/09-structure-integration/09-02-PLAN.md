    **Step 1: Recreate `agents/gsd-research-synthesizer.md`.**

    Base on the Phase 2 specification (documented in research). Key requirements:
    - Frontmatter: `role_type: judge`, `reads: [research-gatherer-outputs]`, `writes: [research-synthesis]`
    - Tools: Read, Write, Bash, Grep, Glob (no external search -- synthesis only)
    - Input: 6 gatherer output files (paths provided by orchestrator) + manifest with success/failed status per gatherer
    - Quality gate: If a gatherer output is < 50 words, mark as failed. If > 3/6 fail, abort synthesis with error.
    - Output: Single RESEARCH.md file at the path specified by orchestrator

    **Output sections** (adapted for planner consumption per research recommendation):
    1. **Consensus** -- findings agreed across multiple gatherers
    2. **Conflicts** -- disagreements between gatherers with resolution reasoning
    3. **Gaps** -- missing information, unfilled dimensions, low-confidence findings
    4. **Constraints Discovered** -- hard limits the planner must respect
    5. **Recommended Scope** -- what to build, what to skip, what to investigate further

    Use `gsd-review-synthesizer.md` as structural reference for the manifest-driven gather-synthesize integration pattern. Both follow the same contract: read gatherer outputs listed in a manifest, consolidate, produce single output file.

    Target: ~400-600 lines, same quality as review synthesizer.

    **Step 2: Create `get-shit-done/workflows/research-workflow.md`.**

    This is a NEW standalone workflow. Model it after `review-phase.md` which also uses gather-synthesize. The workflow must:

    1. Accept inputs via parameters or orchestrator context:
       - `subject`: What is being researched (capability name, feature name, or freeform)
       - `context_paths`: Paths to core context files (PROJECT.md, STATE.md, etc.)
       - `output_dir`: Where to write RESEARCH.md and intermediate gatherer outputs
       - `framing_context` (optional): BRIEF_PATH, LENS, direction, focus (from framing-pipeline)
       - `capability_path` (optional): Path to CAPABILITY.md if scoped to capability
       - `feature_path` (optional): Path to FEATURE.md if scoped to feature

    2. Assemble 4-layer context per gather-synthesize.md contract:
       - Layer 1: PROJECT.md, STATE.md (always)
       - Layer 2: CAPABILITY.md (if capability_path provided)
       - Layer 3: FEATURE.md + REQUIREMENTS.md (if feature_path provided)
       - Layer 4: Framing questions / BRIEF.md (if framing_context provided)

    3. Define the 6 gatherers array:
       ```
       gatherers:
         - agent: agents/gsd-research-domain.md
           dimension: "Domain Truth"
           output: {output_dir}/research/domain-truth-findings.md
         - agent: agents/gsd-research-system.md
           dimension: "Existing System"
           output: {output_dir}/research/existing-system-findings.md
         - agent: agents/gsd-research-intent.md
           dimension: "User Intent"
           output: {output_dir}/research/user-intent-findings.md
         - agent: agents/gsd-research-tech.md
           dimension: "Tech Constraints"
           output: {output_dir}/research/tech-constraints-findings.md
         - agent: agents/gsd-research-edges.md
           dimension: "Edge Cases"
           output: {output_dir}/research/edge-cases-findings.md
         - agent: agents/gsd-research-prior-art.md
           dimension: "Prior Art"
           output: {output_dir}/research/prior-art-findings.md
       ```

    4. Define synthesizer:
       ```
       synthesizer:
         agent: agents/gsd-research-synthesizer.md
         output: {output_dir}/RESEARCH.md
       ```

    5. Invoke gather-synthesize.md with the above configuration.

    6. Return RESEARCH.md path + manifest to caller.

    Use `@~/.claude/get-shit-done/workflows/gather-synthesize.md` as the @file reference for delegation. Use `@~/.claude/agents/gsd-research-*` for agent references.

    **Key design per CONTEXT.md:** "Framing-pipeline determines WHAT to research (scoped by lens). Research workflow determines HOW to research (gatherers + synthesis). Separation enables: multiple callers, independent testability, clean separation of concerns."
