  <name>Task 1: Create plan-validate.cjs with 4-rule validation engine</name>
  <files>get-shit-done/bin/lib/plan-validate.cjs</files>
  <action>
Create `plan-validate.cjs` in `get-shit-done/bin/lib/`. Module exports `cmdPlanValidate(cwd, reqSourcePath, planPaths, raw)`.

**Parameters:**
- `cwd` тАФ working directory root
- `reqSourcePath` тАФ path to FEATURE.md (feature-level) or REQUIREMENTS.md (phase-level) relative to cwd
- `planPaths` тАФ array of PLAN.md file paths relative to cwd
- `raw` тАФ boolean, if true output JSON

**Logic flow:**

1. **Parse REQ source file:**
   - If file contains `| REQ |` or `| REQ ID |` table header: extract REQ IDs from first column of each data row (skip header + separator rows)
   - Regex: `/^\|\s*([A-Z]+-\d+)\s*\|/gm` тАФ captures IDs like EU-01, FN-01, TC-01, PLAN-01, REQS-03
   - Build `validReqIds` Set from matches

2. **Parse each PLAN.md:**
   - Use `extractFrontmatter()` to get frontmatter
   - Extract tasks using regex: `/<task>([\s\S]*?)<\/task>/g`
   - For each task, extract:
     - `<title>` content (for error reporting)
     - `<reqs>` content, split on comma, trim whitespace тЖТ array of REQ IDs
   - Also support v1 format fallback: `<name>` instead of `<title>`, frontmatter `requirements: []` field

3. **Run 4 validation rules:**

   **Rule 1 тАФ Orphan task (ERROR):** Task has empty `<reqs>` or no `<reqs>` tag.
   ```
   { type: 'orphan_task', task: title, plan: filename, message: 'Task has no REQ references' }
   ```

   **Rule 2 тАФ Phantom reference (ERROR):** Task `<reqs>` contains an ID not in `validReqIds`.
   ```
   { type: 'phantom_reference', req: id, task: title, plan: filename, message: 'REQ ID {id} not found in {reqSourcePath}' }
   ```

   **Rule 3 тАФ Cross-layer mixing (ERROR):** Task `<reqs>` contains both EU-xx and TC-xx IDs. Only applies when IDs follow EU/FN/TC pattern (project-level IDs like PLAN-xx, REQS-xx are exempt).
   ```
   { type: 'cross_layer_mixing', task: title, plan: filename, reqs: [ids], message: 'Task mixes EU and TC layers тАФ must bridge through FN' }
   ```

   **Rule 4 тАФ Uncovered REQ (WARNING):** A REQ ID from the source file has no task referencing it in any plan.
   ```
   { type: 'uncovered_req', req: id, message: 'REQ {id} in {reqSourcePath} has no task in any plan' }
   ```

4. **Output:**
   ```javascript
   output({
     passed: errors.length === 0,
     errors: [...],
     warnings: [...],
     summary: { total_tasks, total_reqs, covered, uncovered, errors: errors.length, warnings: warnings.length }
   }, raw);
   ```

**Dependencies:** Require `core.cjs` for `output`, `error`, `safeReadFile`. Require `frontmatter.cjs` for `extractFrontmatter`.

**Key constraint from CONTEXT.md:** "Errors block finalization. Warnings surface in self-critique." The validator only reports тАФ it does not block or finalize anything. That logic lives in the workflow (Plan 03-02).
