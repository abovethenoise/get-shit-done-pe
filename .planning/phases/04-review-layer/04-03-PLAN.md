<!-- Gather-synthesize pattern (reusable) -->
```
Parameters: gatherers[] (agent_path, dimension_name, output_path), synthesizer (agent_path, output_path), context, subject
Step 1: Context Assembly (Layers 0-4)
Step 2: Gather Phase — parallel Task spawns
Step 3: Failure Handling — retry once, abort if >50% failed
Step 4: Synthesize Phase — single agent spawn
Step 5: Completion — return output path + manifest
```

<!-- Review-specific gather-synthesize parameters -->
```
gatherers:
  - agent_path: agents/gsd-review-enduser.md, dimension: "End-User", output: {phase_dir}/review/enduser-trace.md
  - agent_path: agents/gsd-review-functional.md, dimension: "Functional", output: {phase_dir}/review/functional-trace.md
  - agent_path: agents/gsd-review-technical.md, dimension: "Technical", output: {phase_dir}/review/technical-trace.md
  - agent_path: agents/gsd-review-quality.md, dimension: "Code Quality", output: {phase_dir}/review/quality-trace.md
synthesizer:
  - agent_path: agents/gsd-review-synthesizer.md, output: {phase_dir}/review/synthesis.md
```

<!-- Abort threshold (RESEARCH.md) -->
>=2 of 4 failures = abort synthesis (50% threshold from gather-synthesize)

<!-- Q&A pattern (from plan-phase.md step 9.5) -->
Present findings one-at-a-time
AskUserQuestion header max 12 characters (e.g., "Finding 1/7")
5 response options: Accept / Accept w/ Edit (freeform) / Research (freeform) / Defer / Dismiss

<!-- Re-review (CONTEXT.md + RESEARCH.md) -->
Accept or Accept w/ Edit -> re-review affected areas after fixes
Max 2 re-review cycles, then surface remaining for manual resolution
Targeted: only re-spawn affected reviewers + synthesizer always re-runs
Deferred/Dismissed: log in review artifact with label
